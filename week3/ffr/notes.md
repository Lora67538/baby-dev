## 总结
# 核心概念：x * y = k，储备更新、LP 代币铸造/销毁、swap 逻辑
- x，y各自代表一种代币，要求这两种代币在交易前后乘机都得等于z
- 流动池里的代币储备量（即x，y）。
- lp可以按比例存入两种代币x，y；lp销毁lp代币赎回资产时，x，x会减少；用户用x换y时x变多，y就变少了。
- lp在首次部署流动性时，pairs合约会铸造x*y的开方这么多的lp代币；在后续继续存入x，y时会按 存入/目前总价值 的比例计算需要铸造的量。lp想赎回本金和获得手续费时，需要将lp代币销毁。pairs合约根据销毁的 比例给对应数量的两种代币。还能部分销毁？
- 计算好（x+dx）*（y-dy）=z后，执行交易
# 重要函数流程：addLiquidity / removeLiquidity / swap 的参数、顺序、限制
- 三个函数的意思分别是 添加流动性，移除流动性，贷币兑换000000
Factory/Pair/Router 之间的数据流、事件
- F 用户 → Router → Factory（检查交易对是否存在）→ Pair（接收代币、铸LP代币）→ 用户（接收LP代币）
- P 用户 → Router → Pair（销毁LP代币、计算赎回量）→ 用户（接收代币）
- R 用户 → Router → Pair（校验储备、计算兑换量）→ 用户（接收目标代币）
# 手续费计算（0.3% 如何累积、如何返还给 LP）
- 扣除交易的0.30%作为手续费，剩下的才用来计算x*y=z。这0.3%的代币会注入流动池，z值也就会随之变大；手续费又被分成0.25%和0.05%，0.25%归所有lp所有，在赎回时给（以每个lp有的lp代币占总lp代币的比例来分配）；0.05%是协议手续费
